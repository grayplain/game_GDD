# 中世陣取りローグライトデッキビルド ゲームデザインドキュメント

中世〜近世（14〜15世紀）を舞台にした、領土制圧を目的とするローグライトデッキビルドゲームのGDDとバランス検証用シミュレーターの公開リポジトリです。

## ゲームコンセプト

- **ジャンル**：陣取りローグライト × デッキビルド（Steam PC向けインディーゲーム）
- **セッション時間**：約30分
- **勝利条件**：全領土の占領
- **3つの非対称勢力**：軍人（軍事力）・商人（経済力）・宗教家（精神的権威）

## ファイル一覧

### ドキュメント

| ファイル | 説明 |
|---|---|
| `GDD_v1_25.md` | ゲームデザインドキュメント本体。全システム仕様・設計意図・バランス根拠を記載 |

### プロトタイプ

| ファイル | 説明 |
|---|---|
| `strategy_map.html` | 移動フェーズのプロトタイプ。HEXマップ上でユニットを移動させ、戦略マップの基本操作を検証するためのもの |
| `domestic_phase_proto.html` | 内政フェーズのプロトタイプ。HEXマップ・手札・デッキ・AP管理・都市発展などの内政ギミックを検証するためのもの |
| `battle_proto_v1.jsx` | 戦闘フェーズのプロトタイプ。マリガン・AP・士気/HP二重死亡・連鎖崩壊・CPU行動プレビュー・撤退などの戦闘システムを検証するためのもの |

### シミュレーター

| ファイル | 説明 |
|---|---|
| `battle_sim_v2.jsx` | 軍人の戦闘シミュレーター。フォーメーションボーナス（FB）の発動条件と勝率への影響を数値検証するためのもの |
| `battle_sim_intervention.jsx` | 商人の戦闘前介入シミュレーター。介入カードで敵ユニットのステータスをどれだけ削れば商人軍団が太刀打ちできるかを検証するためのもの |
| `formation_sim_v2.jsx` | 軍人のフォーメーションボーナス編成シミュレーター。FB発動パターンと戦闘結果の関係を多角的に検証するためのもの |
| `boss_battle_sim_military.jsx` | 軍人の最終ボス戦シミュレーター。連戦（軍団A→軍団B）でボスを撃破できるかを検証し、ボスのパラメータ調整に使用 |
| `merchant_unit_sim_v7.jsx` | 商人の連戦シミュレーター。介入カードを使いながら軍団A・Bの2連戦でボスを撃破できるかを検証するためのもの |
| `merchant_move_proto.jsx` | 商人の移動・収入フェーズシミュレーター。商人ユニットの交易ルートと収入獲得の挙動を検証するためのもの |
| `fb_montecarlo_sim_v2.html` | 軍人のフォーメーションボーナス編成モンテカルロシミュレーター。25ターン以内に目標編成を達成できる確率をランダムサンプリングで推定するためのもの |
| `military_hire_sim.html` | 軍人のユニット雇用シミュレーター。ショップ提示数・レアリティ分布・金収支をもとに目標ユニット構成の達成確率を検証するためのもの |

## 技術スタック

- プロトタイプ（HTML）：Vanilla JS、単一ファイルでブラウザ直接起動
- シミュレーター（JSX）：React（Artifactまたはローカル環境で動作）
- デザインテーマ：ダーク中世風（Cinzel / Crimson Textフォント、ダークブルー/ゴールド配色）
