# **7\. 内政システム**

## **7.0 内政システムの設計方針　★v1.7新規追加**

内政フェーズは「戦闘のための準備フェーズ」と位置づける。内政そのものを楽しむゲームではなく、戦闘に向けた読み合いと選択を行うフェーズ。

**採用する2つのコンセプト**

| コンセプト | 内容 | 採用理由 |
| :---- | :---- | :---- |
| カードドラフト・選択イベント | 毎ターン複数の選択肢から1つを選ぶ。「何を取るか」の判断がフォーメーション役を狙う戦略に直結する | ラン毎に狙う役が変わる→飽きにくい。敵編成を読んだ上での選択がドラフトの読み合いを生む |
| 領地ごとの特性活用 | 各HEXが固有ボーナスを持ち、どの領地を優先発展させるかで戦略の形が変わる | 史実的根拠が高い（中世の地政学）。HEX占領の意思決定に奥行きを加える |

*※ エンジンビルド・リソース変換チェーンは30分〜1時間のセッション長・「内政は準備フェーズ」のコンセプトと相性が悪いため不採用。*

*※ 史実的根拠：中世君主の「限られた勅令枠」に近い構造。動員令・課税・宗教的布告のうち何を優先するかの政治判断が、ドラフトの選択として表現される。*

**詳細設計は今後検討**（現在のカード設計は既存の内政カード設計を継続使用、ドラフト・領地特性の具体的な実装は別途設計）

### **9.0.1 設計方針**

フォーメーションボーナス（FB）システムを廃止し、軍人固有システムとして「軍団フローチャートシステム」を導入する。

**コアコンセプト：事前に組んだロジックが想定通りに動く達成感**

| 項目 | 内容 |
| :---- | :---- |
| ユニット | フローチャートの1記号（ノード）として機能する |
| 配置順序 | そのままフローの実行順序になる |
| 役割 | 各ユニットは2〜3種のFC記号役割を持ち、内政フェーズで切り替え可能 |

*※ 史実的根拠：中世の指揮官（コンデスタブルなど）は事前に複数のシナリオを用意し、斥候報告に基づいて実行計画を切り替えた。アジャンクールではヘンリー5世が地形・泥地・狭路という条件を事前に読んで弓兵主体のフローを固定し、開戦後は計画通りに実行するだけだった。「条件を事前に定義して実行を委ねる」は中世指揮構造と高い整合性を持つ。*

### **9.0.2 FC記号種別一覧（検討案・要プロトタイプ検証）**

各記号の具体的な効果数値はプロトタイプで検証後に確定する。以下は設計方針として記録する。

| 記号種別 | フローチャート上の役割 | 設計方針 | 候補ユニット |
| :---- | :---- | :---- | :---- |
| 処理 | 値を生産・加算する | フローに攻撃値を供給する基本ノード | 槍兵・各ユニットの基本役割 |
| 判断 | 変数を見て実行経路を分岐する | 条件カードをセットして分岐条件を定義 | 司令官（主担当） |
| 増幅 | 前ノードから受け取った値を乗算する | 単体では機能しない。前段に「処理」が必要 | 騎士 |
| 遅延 | 値を次ラウンドに持ち越す | 溜めた値を後段の増幅と組み合わせることで爆発力が生まれる | 重装歩兵 |
| 割り込み | フロー順序を無視して先行実行する | 敵フローより先に処理を走らせる | 騎馬兵 |
| スキップ | 前衛を飛ばして後衛に作用する | 対象選択のメタ操作 | 弓兵・騎馬兵 |
| 出力 | 蓄積した値をダメージとして確定する | 出力ノードがなければ蓄積値はダメージにならない | 弓兵・弩兵 |
| 貫通 | 防御フラグを無視して実行する | アーマー・シールド等を無効化 | 弩兵 |
| 参照 | 敵変数を読んで後続ノードに渡す | 斥候なしでは参照できない敵情報を取得 | 斥候・司令官 |
| バッファ | 値を溜めて後続に一括放出する | 単独では小さいが複数の後段に同時供給できる | 槍兵 |
| 繰り返し | 同じ処理を指定回数実行する | ループ先・上限回数は未定。要検証 | 騎士 |

*※ 設計原則：上位互換を排除する。重装歩兵と騎士だけ並べてもダメージ最大にはならない。「値を生む」「値を増幅する」「値を出力する」の役割が揃って初めて掛け算的効果が発動する分業構造を目指す。*

*※ 史実的根拠：中世軍の分業は明確だった。長弓兵は「ダメージ出力」、騎士は「突撃による乗算的衝撃力」、歩兵は「陣形維持（バッファ）」、指揮官は「状況判断と作戦変更（判断）」。各兵種が単独では機能せず組み合わせで初めて戦術が成立する点が、フローチャートの分業構造と対応する。*

### **9.0.7 敵ユニット設計（兵種・カウンター能力・ターゲット選択）　★v1.34新設**

#### **敵兵種定義**

敵ユニットは3種の兵種に集約する（三すくみ構造）。プレイヤー側の10種より大幅に絞ることで実装コストを抑えつつ、兵種相性のゲーム性を実現する。

| 敵兵種 | カウンター対象 | 優先ターゲット |
| :---- | :---- | :---- |
| 槍兵 | 騎馬兵ノード | 前衛ユニット（現行踏襲） |
| 弓兵 | 槍兵ノード | HP最低のユニット |
| 騎馬兵 | 重装歩兵ノード（★v1.39変更） | 雇用コスト最高のユニット |

*※ 三すくみ構造の史実的根拠：「槍→騎馬→弓→槍」の相性は中世ヨーロッパで繰り返し観察される構造。スイス傭兵の槍衾がブルゴーニュ騎士団を壊滅（1476年）、アジャンクールで密集歩兵が弓兵の格好の的になり（1415年）、騎馬が弓兵・投石兵などの機動力のない遠距離ユニットを踏み潰す役割を担った。*

#### **カウンター能力**

プレイヤーが対象ユニットのノードを実行した瞬間、対応する敵兵種が割り込んで発動する。カウンターのHP直接減算は「即時ダメージ（INSTANT）」出力種別として正式設計（★v1.35確定）。

**★v1.38変更：カウンター効果は「発動したユニットへのHP即時ダメージ」のみとする。蓄積ATK（state.atk）への干渉（攻撃力低下デバフ）は廃止。**

| 敵兵種 | 発動条件 | 効果 | 史実的根拠 |
| :---- | :---- | :---- | :---- |
| 敵槍兵 | 騎馬兵ノード実行時 | 騎馬兵ユニットにHP即時ダメージ（INSTANT） | スイス槍衾は騎馬に刺さり騎手がダメージを受ける。しかし騎馬の突進力（蓄積された運動量）そのものは槍衾に届く |
| 敵弓兵 | 槍兵ノード実行時 | 槍兵ユニットにHP即時ダメージ（INSTANT） | 矢は槍兵に刺さる。しかし槍兵がそれまでに積み上げた前進の勢いは止まらない（アジャンクールで矢を受けながら前進した記録あり） |
| 敵騎馬兵 | 重装歩兵ノード実行時（★v1.39変更） | 重装歩兵ユニットにHP即時ダメージ（INSTANT） | ヘイスティングス（1066）でノルマン騎兵がアングロサクソンのシールドウォール（重装歩兵密集陣形）の側面を突いて崩壊させた。重装歩兵は正面には強いが側面・背面への対応が遅い |

*※ 数値（追加ダメージ量）はすべて暫定。プロトタイプ検証後に調整予定。*
*※ v1.38廃止：atkDebuffによるstate.atk直接減算。史実的根拠：迎撃（カウンター）は攻撃行為者にダメージを返すものであり、「相手が積み上げた攻撃の勢い・蓄積値を消去する」という効果は史実に対応物がない。蓄積ATKが届くかどうかはフロー設計（出力ノードの有無）で決まる構造とする。*
*※ 弓兵カウンター発動上限（★v1.39変更）：従来の固定3回/戦闘から「敵弓兵数×3回/戦闘」に変更。敵弓兵が1体なら3回、2体なら6回。史実根拠：矢の備蓄は弓兵の人数に比例する。アジャンクールでイングランド軍が密集フランス軍に大量射撃できたのは長弓兵を大量配備していたから。*
*※ 近接反撃ギミック（★v1.39新設）：プレイヤー前衛ユニット（槍兵・盾兵・重装歩兵・騎馬兵・騎士）が攻撃した際、攻撃対象の敵が近接兵種（槍兵・騎馬兵）なら攻撃したユニットに反撃ダメージを返す（パターンA）。敵近接ユニットがプレイヤー前衛を攻撃した際、攻撃を受けたプレイヤー前衛ユニットも即時反撃ダメージを返す（パターンB）。反撃ダメージは固定値（暫定）。騎士のみアーマーで被反撃を無効化。史実根拠：中世白兵戦は接触時に双方が消耗する。槍衾同士のぶつかり合いは前列が確実に傷つく押し込み戦術であり、一方的な攻撃は発生しない。*
*※ deadNodeSkipデフォルト変更（★v1.39）：死亡ユニットのノードスキップをデフォルトONに変更。死亡ユニットのノードは補給を消費せずスキップされる。生存ユニット数に応じて補給初期値・ループ開始時補給も連動して減少する。「フローは止まらない」はオプション扱い。*

#### **優先ターゲット選択**

各敵兵種はラウンド開始時に攻撃対象を選択する。優先対象が存在しない場合（全員撃破済みなど）は前衛から順に攻撃する。

| 敵兵種 | 優先ターゲット | 設計意図 | 史実的根拠 |
| :---- | :---- | :---- | :---- |
| 槍兵 | 前衛ユニット | 現行の固定攻撃を踏襲。正面の敵を押しとどめる盾の役割 | 槍衾は正面突破を防ぐ壁として機能。特定ユニットを狙うより前進阻止が目的 |
| 弓兵 | HP最低のユニット | 脆いユニットを確実に仕留めてコスト効率を最大化 | 中世の弓兵・弩兵は負傷兵・疲弊した兵を優先して狙い撃ちした（確実な排除でAP効率を高める） |
| 騎馬兵 | 雇用コスト最高のユニット | 精鋭・高価値ユニットを潰して戦意を崩す | 中世の騎馬突撃は敵の指揮官・騎士など高価値ターゲットを優先した。首を取れば敵が崩壊する |

*※ 設計意図：昨日のメモの「HP最低 or 高価なユニットを優先狙い」に兵種別の個性を組み合わせた。これにより「弓兵がいる敵陣では低HPユニットを守る防御戦略」「騎馬兵がいる敵陣では高コストユニットを守る均等育成戦略」という戦略分岐が生まれる。*

#### **敵編成パターン（4型）　★v1.41新設**

## **9.4 士気・HPルール　★v1.0全面改訂・v1.33フローチャートシステム対応刷新**

**【設計方針】士気は「閾値ゲーム」として機能させる**

**士気は線形に弱体化するのではなく、ある閾値を下回った瞬間に連鎖崩壊が起きる設計とする。**

史実（クレシーの戦いなど）でも、軍は閾値を超えた瞬間に雪崩式に崩壊するパターンが常態だった。

\-1程度の士気削りでは大きな効果はないが、ある水準（士気3〜4付近）まで下がると離脱連鎖が発生しやすくなる。

この閾値をプレイヤーが読み合う設計にするため、全ユニットの現在士気はUIで常時可視化する。

**【v1.33 重要変更】ATKダメージによる士気減少を廃止**

フローチャートシステムとの整合性から、通常攻撃を受けても士気は減少しない。士気を削る手段は専用の士気攻撃出力のみとする。これにより「ATKで殴り続けるのが最適解」という問題を解消し、士気攻撃を意識的な戦略選択として成立させる。

*※ 史実的根拠：中世戦場の死傷者の主因は弓・槍であり、騎馬突撃は心理的恐怖（士気崩壊）が主効果だった。ATKと士気を別軸にすることで、「重装歩兵はATKで削りにくいが士気が低いので崩しやすい」という案Aの読み合いが生まれる。*

**士気の減少源**

| 減少源 | 士気減少量 | 備考 |
| :---- | :---- | :---- |
| ~~通常攻撃を受ける~~ | ~~-1（固定）~~ | ~~★v1.33廃止：ATKと士気を分離~~ |
| 士気出力ノード（フロー） | -N（ノード設定値） | 騎馬兵コア能力(-3)など。フロー実行時に適用 |
| 仲間がHP死亡 | \-2 | 前衛崩壊→連鎖崩壊の核。閾値を下回るトリガーになりやすい |
| 包囲状態 | \-2/ラウンド | 継続的なプレッシャー |

**士気の回復源**

| 回復源 | 士気回復量 | 備考 |
| :---- | :---- | :---- |
| 司令官（常駐中） | \+2回復/ラウンド | 削る vs 回復の綱引きが発生。閾値を維持するカウンター役 |
| 士気回復カード | カード記載通り | 内政フェーズ・戦闘フェーズで使用 |

**士気0・HP0の処理**

| 状態 | 処理 | 備考 |
| :---- | :---- | :---- |
| 士気0 | 即離脱（戦闘から退場） | HPはそのまま残る・内政フェーズで回復可能 |
| HP0 | 永久消滅 | ゲームから除外 |

*※ 史実同様、全員戦死より士気崩壊による撤退で決着がつくケースが大半になる設計。HP0は例外的な消滅。*

## **9.4.1 連戦システム（戦闘間回復）　★v1.12新規追加**

**【設計方針】1ターン内に複数軍団を順番に進軍させ、消耗を蓄積させて敵を崩す**

移動フェーズにおいて、同一ターン内に複数の自軍軍団を同一の敵HEXへ順番に進軍させることができる。最初の軍団で敵を消耗させ、次の軍団でとどめを刺す連戦戦術が成立する。

**軍団の定義**

| 項目 | 内容 |
| :---- | :---- |
| 構成 | 1〜6ユニットで編成されたグループ |
| 単位 | 1HEXに配置されたユニット群がそのまま1軍団として進軍する |

**戦闘間回復ルール**

1回の戦闘が終了し、次の軍団が進軍してくるまでの間に、防衛側の残存ユニットは以下の通り自動回復する。

| 回復対象 | 回復量 | 備考 |
| :---- | :---- | :---- |
| HP | 最大HPの30%（端数切り捨て） | 残存ユニットのみ |
| 士気 | 最大士気の30%（端数切り捨て） | 残存ユニットのみ |
| 離脱ユニット | 回復・復帰なし | 士気0で離脱したユニットはその戦闘限りで不参加のまま |

*※ 「残存ユニット」とは、士気0による離脱・HP0による消滅のいずれも発生していないユニットを指す。*

*※ 攻撃側（進軍した軍団）は戦闘間回復なし。第1軍団は消耗したまま次の戦闘には参加しない（使い捨て前提）。数値調整はシミュレーション後に確定予定。*

**連戦の戦術的意図**

| 戦術 | 内容 |
| :---- | :---- |
| 先鋒消耗 | 第1軍団で敵のHP・士気を削り（例：最大値の約40%消耗）、第2軍団でとどめを刺す |
| 有利交換 | 第1軍団が捨て駒として敵の強力なユニットを士気崩壊させ、第2軍団が数的優位で戦う |
| 連続圧力 | 3割回復があっても2連戦の累積消耗は大きく、完全回復しないまま第2軍団に当たることになる |

*※ 史実的根拠：中世の城攻めでは「第一波で守備隊を疲弊させ、第二波で突入する」波状攻撃が常套手段だった。クレルモン公会議（1095年）以降の十字軍も、都市攻略に際して複数の攻撃波を交互に投入する手法を採用している。また、攻城側が複数梯団を用意して消耗を補填するのに対し、守備側は3割程度の体勢立て直し（負傷兵の応急手当・隊列再編）は可能でも完全回復はできないという非対称性が、攻城の優位性を生んでいた。*

## **9.X 調略システム　★v1.0新規追加**

**【設計方針】軍人は戦場で強い。商人・宗教家は戦場に持ち込ませない。**

**正面戦闘では軍人が優勢。商人・宗教家はこの非対称性を前提に、戦闘前に軍人の戦力を削ることでバランスを取る。**

## **9.6 撤退システム \[確定\]**

撤退は戦闘カードではなく、専用の撤退ボタンで実現する。

| 項目 | 内容 |
| :---- | :---- |
| 撤退タイミング | 戦闘中、自分のカードプレイフェーズならいつでも可 |
| 操作方法 | 撤退ボタンを押すと即座に戦闘終了 |
| ユニット温存 | HP残存ユニットは全員温存（離脱状態ユニットも含む） |
| 士気ペナルティ | 全員に士気-3（撤退の代償） |

## **13.3 守備隊の毎ターン強化 \[確定\]**

プレイヤーが戦闘を先延ばしにするほど守備隊が強くなる「攻城戦のプレッシャー」を再現する。

| ターン帯 | 強化内容 |
| :---- | :---- |

**最終ボスの扱い：**

最終ボス（HEX 7）はターン経過による強化を受けない。開始から終了まで固定性能。

*※ 設計意図：最終ボスは「到達したときの実力勝負」として設計する。ターン経過で強化されると「早く行かなければ」という焦りが生まれすぎてプレイヤーの準備行動を否定してしまう。中間守備隊の強化は「寄り道コスト」として機能し、ボスは「ゴールとしての安定した挑戦」として機能する。*

*※ 史実的根拠：守備側は時間があれば補給・士気立て直し・援軍要請ができた（士気回復・ATK/HP上昇）。また長期放置された地域には「流民・傭兵崩れの合流」により守備兵力が増えるケースがあった（ユニット数増加）。*

## **13.5 襲撃イベント \[確定\]　★v1.27全勢力共通化**

### 概要

### 発生頻度

| 項目 | 内容 |
| :---- | :---- |
| 基本頻度 | 5〜7ターンに1回（ランダム幅） |

*※ 設計意図：各勢力の固有リソースを貯めることに「襲撃を少し抑制する」副次的価値を与える。リソース管理の選択肢が広がる。*

| 項目 | 内容 |
| :---- | :---- |

*※ 史実的根拠：交易路・経済拠点の奪還は最も合理的な反撃目標だった。無人地帯への侵攻は中世で最も一般的な領土回復手段（略奪・接収）。新占領地は防衛体制が整っていないため反撃されやすい。*

# **16\. 商人固有システム：襲撃イベント・最終決済　★v1.13新規追加**

## **16.1 設計方針**

商人勢力のゴールは「HEX7を軍事占領する」ことではなく、**「最終的な大波を生き延びること」**である。

ユニットの役割は「攻める」ことではなく「守る」こと。傭兵軍を保持する目的は積極攻勢ではなく、略奪・封建領主の侵攻からネットワークを守るためである。

**★v1.19 収入設計の根本変更**

商人の収入は「動かすことで稼ぐ」フロー型に統一する。都市収入（毎ターン自動のストック型）は商人には適用されない。

| 収入軸 | 仕組み | プレイヤーの判断 |
| :---- | :---- | :---- |
| 内政ユニット移動 | 毎ターン市況の良いHEXに配置して稼ぐ（18章） | どのHEXに誰を送るか |
| 投資イベント | 盤面を読んでハイ/ローリスクを選ぶ | 敵編成・残りターンから期待値を推測 |
| 内政カードプレイ | 交易収入カードをプレイして即時獲得 | APの使い道をどこに割くか |

*※ 史実的根拠：ハンザ同盟・ヴェネツィア共和国・フッガー家はいずれもフロー収入型。都市収入（地代・農奴税）は封建領主の収入モデルであり、商人とは根本的に異なる。「動かさないと稼げない」設計が商人の本質を最もよく再現する。*

*※ 史実的根拠：ヴェネツィア共和国・ハンザ同盟は領土征服を目的とせず、商業ネットワークと都市を守り続けることを国家目標とした。「最終的な大波を生き延びる」が商人の勝利であるのは、中世商人都市国家の実態に最も近い設計である。*

## **16.3 最終決済イベント（ターン25）\[確定\]**

### **16.3.1 概要**

ターン25の内政フェーズ開始時に「最終決済」イベントが発生する。これが商人の実質的なラスボス戦である。

| 項目 | 内容 |
| :---- | :---- |
| 発生タイミング | ターン25 内政フェーズ開始時 |
| イベント名 | 最終決済（Final Settlement） |

| 事前通知 | ターン24の開始時に「次のターン、最終決済が迫っています」と告知 |

### **16.3.2 解決方法（2択）**

プレイヤーは内政フェーズ中に以下の2択で最終決済に対応する。

**選択肢①：金による回避（買収解決）**

| 項目 | 内容 |
| :---- | :---- |
| 仕組み | 規定の金額を支払うことで侵攻軍を引き上げさせる |
| コスト | 相手勢力の現在戦力×係数（未定）。弱い相手ほど安く、強い相手ほど高い |
| 成功条件 | 規定金額を全額支払える場合のみ選択可能 |
| 結果 | **商人の勝利** |

*※ 史実的根拠：フッガー家はカール5世の戦費を融資することで政治的影響力を行使し、敵対勢力を「金で黙らせる」外交を常套手段とした。最終決済の「金で回避」はこの史実に直接対応する。*

**選択肢②：防衛戦（軍事解決）**

金が不足する場合、または意図的に戦闘を選ぶ場合の選択肢。

| 項目 | 内容 |
| :---- | :---- |
| 仕組み | 侵攻軍との防衛戦を行う |
| 勝利条件 | 侵攻側を完全に全滅させること（生存者1体でも残れば敗北） |
| 金の事前投入 | 防衛戦前に手持ちの金を全額使って戦力を削ることができる（16.3.3参照） |
| 敗北条件 | 侵攻側が1体でも生き残る |
| 結果（勝利） | **商人の勝利** |
| 結果（敗北） | ゲームオーバー |

*※ 完全勝利条件（侵攻側全滅）を採用する理由：最終戦なので「辛勝でも勝ち」ではなく、金を使い切っても勝てないなら経済戦略の失敗として一貫した設計にする。*

### **16.3.3 防衛戦前の金投入（戦力削減）**

防衛戦を選択した場合、戦闘前に保有金を消費して侵攻軍の戦力を削ることができる。

| 金投入量 | 効果 |
| :---- | :---- |
| 金5〜9 | 侵攻軍の前衛1体を離脱させる |
| 金10〜14 | 侵攻軍の前衛2体を離脱させる |
| 金15以上 | 侵攻軍の前衛2体を離脱＋全体士気-3 |

*※ 設計意図：金が多ければ回避できるが、中途半端な金額では「回避できないが防衛戦を有利にする」という使い道が生まれる。金を「全部使って回避する」か「一部使って戦いを有利にして防衛に賭ける」かの判断がジレンマを生む。*

*※ 数値は未確定。シミュレーション検証後に調整予定。*

# **17. 交易都市システム　★v1.17追加**

## **17.1 概要 \[確定\]**

毎プレイのマップ生成時に、通常HEXとは異なる特殊HEX「交易都市」が1〜2箇所配置される。交易都市は全勢力が占領する価値を持つが、受ける恩恵の大きさは勢力によって異なる。

| 項目 | 内容 |
| :---- | :---- |

| 配置位置 | マップ生成時にランダム決定（HEX7とは別のHEX） |
| 位置づけ | HEX7（最終ボス）とは独立した中間目標 |
| 競合設計 | プレイヤー・CPUともに優先的に狙う対象として設計する |

*※ 設計意図：商人がマップフェーズで積極的に動くインセンティブを作ると同時に、全勢力が争う争点としてマップ上の緊張感を高める。*

*※ 史実的根拠：中世の交易都市（フランドル・シャンパーニュ・ハンザ都市）は複数勢力が支配権を争った実際の争点だった。同じ都市でも封建領主には徴税権、商人には関税収入、聖職者には巡礼収入と、立場によって得るものが異なっていた。*

## **17.2 占領効果 \[確定\]**

交易都市を占領すると、占領している間だけ以下の効果が発生する（**占領維持型**）。占領を失った時点で効果は消滅する。

| 効果カテゴリ | 具体的な効果 | 主な受益勢力 |
| :---- | :---- | :---- |
| 金収入増加 | 内政フェーズ開始時に金+2 | 商人（最も恩恵） |
| 信者数増加 | 内政フェーズ開始時に信者数+1 | 宗教家 |
| ドラフト強化 | 内政カードのドラフト選択肢+1枚 | 全勢力共通 |

*※ 効果の付与組み合わせ：毎プレイのマップ生成時に上記から2〜3種がランダムに選ばれる（固定にするか都市ごとに変えるかは未定）。*

*※ 占領維持型を採用する理由：「取れたらラッキー」ではなく「守り続ける」緊張感を生み出すため。奪われた瞬間に恩恵が消えることで、マップフェーズ全体を通じた争点として機能する。史実でもヴェネツィアがコンスタンティノープルの商館を失った後に東方交易が壊滅したように、交易都市は「維持してこそ意味がある」存在だった。*

## **17.3 商人への特別連動：最終決済コスト低減 \[確定\]**

| 状態 | 最終決済の金コスト |

| 交易都市を2箇所占領中 | 通常コスト × 0.6（仮値・要調整） |

*※ 設計意図：交易都市を取りに行く動機が「金収入の増加（短期）」と「最終決済コストの低減（長期）」の二重構造になることで、中間目標としての機能強度を確保する。*

*※ 「占領維持」が条件なので、ターン25直前に奪われると割引が消える。守り続けることへの緊張感がゲーム終盤まで持続する。*

*※ 史実的根拠：ハンザ同盟が各地の交易拠点（コントール）を維持することで交易コストを抑制し競合他者を排除していた。交易拠点の喪失は直接的な経済的損失として跳ね返っていた。*

# **18. 交易市況システム　★v1.19新設・v1.20全面更新**

## **18.1 概要**

## **18.2 市況の種類と分布 \[確定\]**

各HEXの市況は毎ターン完全ランダムで決定される。

| 市況 | 収入レンジ | 空振り確率 | 史実イメージ | 分布（10HEX中） |
| :---- | :---- | :---- | :---- | :---- |
| 好況📈 | 金+4〜6 | 0% | 豊作・航路開通・需要急増 | 2HEX |
| 普通🏪 | 金+2〜3 | 0% | 平時の定常交易 | 4HEX |
| 不況📉 | 金+1 | 0% | 凶作・競合増加・需要低迷 | 2HEX |
| 危険⚠️ | 金+6〜8 or 0 | 45% | 海賊・略奪・政変リスクを含む高利交易路 | 2HEX |

*※ 数値はモンテカルロシミュレーション（merchant_unit_sim_v7）で検証済み。デフォルト値のまま以下の部分情報公開設計と組み合わせてバランスを確認している。*

*※ 分布設計の意図：好況と危険は各2HEXで希少。5体のユニットを配置する場合、「全員好況を狙う」ことは現実的でなく、必ず分散配置の判断が生まれる。*

## **18.3 部分情報公開システム \[確定\]　★v1.20新設**

毎ターン、全10HEXのうち**2HEXのみ市況が公開**される。残り8HEXは市況が不明のまま配置判断をおこなう。

| 項目 | 内容 |
| :---- | :---- |
| 公開数 | 毎ターン2HEX（ランダム選択） |
| 非公開HEXの扱い | 市況が表示されない。プレイヤーは過去の傾向・ユニット特性から推測する |
| 判断の価値（シミュレーション検証） | 公開情報を活かして最適配置した場合、ランダム配置比で**+35%の収入向上**（500試行×25ターン） |

*※ 設計意図：「2HEXだけ見える」状態では毎ターン10個中8個が霧の中であり、得た情報を誰に使わせるかの判断が毎ターン生まれる。情報を活かせば有意に稼げるが、情報がない状況でも「ユニット特性を信じた配置」で安定収入を確保できる設計。*

*※ 判断の価値+35%の根拠：ランダム配置（下限）と全情報最適配置（理論上限）の中間に、部分情報最適が収まっている。「情報を活かすとクリアできる、無視するとキツい」というローグライト的ゲームバランスに合致する。*

*※ 史実的根拠：中世商人の情報収集は常に不完全だった。ハンザ同盟の商人は信書や口伝で各地の相場を得ていたが、遅延・欠落が常態であり「すべての市況を完全に把握して交易する」ことは不可能だった。2HEXのみ公開という設計は、中世商人が「断片的な情報を手がかりに判断を下す」実態を再現している。*

## **18.4 HEX別リスク・リターン設計 \[確定\]**

| 中立HEX（検討中） | 自国と敵国の中間。バランス型 |

*※ 史実的根拠：ハンザ商人がイングランド王から商権制限を受けたり、ノヴゴロドで商館閉鎖されたりという「敵地ビジネスのリスク」が実際に存在した。敵地は高収益だが不安定という非対称設計は史実に対応する。*

*※ 中立HEXは検討段階。「誰の土地かより、そこで商売できるかどうか」が商人の判断基準という史実の論理に合致するが、ゲームバランスの観点から実装優先度は中程度。*

| ユニット | 固有特性 | 推奨配置先 |

*※ 密輸商人の自国HEXペナルティ（-1）の設計意図：「密輸は自国では目立つ」という設定。安全地帯に逃げることへの抑止として機能し、自然と危険HEXへ向かう動機付けになる。*

*※ 両替商の保証値設計の意図：不況や危険では「損しない」ことを保証し、上振れは諦める代わりにリスクヘッジとして機能する。中世の両替商（カンビスタ）は為替リスクを吸収して手数料を取るビジネスモデルであり、「最低保証」という特性は史実の機能に対応する。*

## **18.7 同HEX相互作用システム \[確定\]　★v1.20新設**

| 在地商人＋両替商 | 在地商人のボーナス+1・両替商の最低保証+1（普通/不況HEXのみ有効） | 同一HEXに共存かつ普通/不況市況 |

*※ 設計意図：「誰と誰を同じHEXに送るか」という2次元の判断が毎ターン生まれる。市況だけでなくユニット同士の相性を読むことが上位プレイに繋がる。*

*※ 史実的根拠：中世の金融取引において、両替商（カンビスタ）と仲介商人（センサーリョ）は同じ取引所・市場に共存することで互いの業務効率を高めていた。シャンパーニュ大市では仲介人と両替商が物理的に隣接して業務を行っており、「同居することで相乗効果が出る」設計は史実の市場構造に対応する。*

# **19. カード取得システム　★v1.26新設**

## **19.1 設計方針 \[確定\]**

カード取得の主軸は**毎ターンのドラフト**（Monster Train型）とし、サブ経路としてショップとイベントを設ける。

| 項目 | 内容 |
| :---- | :---- |
| 主軸 | 毎ターンのドラフト（内政フェーズ中に実施） |
| プール | 勢力別（自勢力のカードしか出ない） |
| 提示枚数 | 軍人：2枚提示→1枚選択 or スキップ　／　商人・宗教家：3枚提示→1枚選択 or スキップ |
| スキップ | 常時可能。「取らない選択」はデッキの質を保つ重要な判断 |
| デッキ上限 | **なし**。枚数管理はプレイヤーの判断に委ねる |
| カード種混在 | 内政・調略・戦闘カードが毎ターン混在して提示される |

*※ 設計意図：「どのカードを取るか」だけでなく「取らない判断」も重要。デッキが肥大化すると引きたいカードを引けない確率が上がるため、上限を設けなくとも自然な抑制が働く。StS的な「デッキ圧縮の価値」をターン帯の選択肢変化と組み合わせて実現する。*

## **19.2 フェーズ連動型ドラフト \[確定\]**

ターン帯によって提示されるカードの**出現比率が自動シフト**する。プレイヤーの意識が自然に「内政→工作→決戦」へと移行するゲームリズムを生む。

| ターン帯 | 内政カード | 調略カード | 戦闘カード | 史実イメージ |
| :---- | :---- | :---- | :---- | :---- |
| T1〜8（序盤） | 65% | 20% | 15% | 領地経営・徴募期。戦闘カードは稀少で取るか迷う選択になる |
| T9〜16（中盤） | 40% | 35% | 25% | 隣国との競合が本格化。工作・調略が主役に |
| T17〜25（終盤） | 20% | 25% | 55% | 決戦前夜・総動員。戦闘カードが主役に転換 |

*※ 史実的根拠：中世の戦役は春〜秋の「戦争シーズン」に集中し、冬〜春は領地経営と徴募に費やした。百年戦争でもエドワード3世は序盤を補給・同盟形成に費やし、クレシー（中盤相当）以降に大規模決戦へ転じた。ターン帯でカード比率が変わる設計は「内政→工作→決戦」という史実の戦役サイクルを自然に再現する。*

*※ 軍人のドラフト提示枚数は**2枚**（他勢力は3枚）。軍人には調略カードプールが存在しないため、3枚提示にすると「使えないカードが混じる」ノイズ感が生じる。最初から2枚提示とすることで「軍人はシンプルな判断をする」という勢力コンセプトを体験レベルで表現する。史実的根拠：封建領主の意思決定は「戦うか・備えるか」の2〜3択が基本であり、商人や聖職者のような複雑な外交選択肢を常時持っていたわけではなかった。*

## **19.3 ショップ \[確定\]**

固定ターンに登場するショップで金を消費してカードを購入できる。ドラフトで取り逃したカードの補完、または戦略的に欲しいカードを確実に入手する経路。

| T20（終盤） | 4枚 | 内政10% / 調略20% / 戦闘70% | 金3〜5 |

*※ T20ショップは「最終決戦前の最後の補強」として機能する。戦闘カードが主役になる。*

*※ 購入は任意。1回のショップで複数枚購入も可（金が続く限り）。スキップも可。*

*※ 史実的根拠：中世の傭兵・武器調達は戦役前の「兵站準備期間」に集中した。遠征前に武器商人・傭兵斡旋業者を呼んで必要な戦力を調達するのが封建軍の常態だった。固定タイミングのショップは「出陣前の最終調達」という史実の準備フェーズを表現する。*

# **20. 軍団フローチャートシステム詳細　★v1.30新設**

## **20.1 システム概要**

| :---- | :---- | :---- |

| 役割切り替えタイミング | 内政フェーズのみ | **確定** |

## **20.2 フロー定義の手順**

1. 内政フェーズに各ユニットのFC記号役割を選択（2〜3択）
2. 司令官がいる場合、条件カード（専用別枠）を1枚セット
3. 移動フェーズでユニットをHEXに配置（配置順 = フロー実行順）
4. 戦闘フェーズ開始時にフローが固定される
5. 毎ラウンド先頭ノードから順に実行

## **20.3 FC記号種別と設計方針**

詳細は9.0.2を参照。効果の数値・相互作用の具体構造はプロトタイプ検証後に確定する。

**設計の核心：分業による相互依存**

```
[処理ノード（槍兵）] → 攻撃値Nを生産
         ↓
[増幅ノード（騎士）] → 受け取った値を×2
         ↓
[出力ノード（弓兵）] → 蓄積値をダメージとして確定
```

処理ノードなしに増幅ノードを置いても増幅する値がない。出力ノードなしに増幅しても蓄積値はダメージにならない。この相互依存がフローの「組み合わせ」に意味を与える。

## **20.7 フローの出力リソース　★v1.30追加・v1.31更新・v1.35拡張・v1.38方針刷新**

フロー実行によって得られる出力は5種類とする。既存のユニットパラメータ（攻撃力・士気）はフローの入力値としては使用しない。

**★v1.38基本方針：フローの基本出力はINSTANT（即時ダメージ）。ATK蓄積型は増幅チェーン（処理ノード→増幅ノード→END）を意図的に組んだ場合の例外的戦術として位置づける。**

| 出力種別 | 意味 | 確定タイミング | 備考 |
| :---- | :---- | :---- | :---- |
| 攻撃力 | 敵ユニットに与えるダメージ | ループEND時に一括確定・リセット | 蓄積型（増幅チェーン専用） |
| 防御力 | 敵から受けるダメージの軽減値 | ループEND時に一括確定・リセット | 蓄積型 |
| 補給 | ノードを実行できる回数リソース | 消費・獲得はループをまたいで継続 | 持続型 |
| 士気 | 敵ユニットの士気を削る | ループEND時に一括確定・リセット | 蓄積型 |
| **即時ダメージ（INSTANT）** | **ノード実行時点でHP直接ダメージ** | **ノード実行と同時に即時確定** | **★v1.35新設・v1.38基本出力** |

*※ v1.38出力方針の設計根拠：中世白兵戦・弩・騎馬突撃はいずれも接触時に即時ダメージが発生する。「貯めてから放つ」運用が成立するのは長弓の斉射（volley fire）のみ。この史実構造に対応して、ゲーム上の基本出力もINSTANTとし、ATK蓄積型は増幅チェーンを明示的に組んだ場合の高リターン・高リスク戦術に限定する。テストプレイで「基本は即時の方が直感的」という知見から採用（v1.38）。*

*※ 即時ダメージの設計根拠：蓄積型の攻撃力はEND時に前衛へ一括ダメージを与えるのに対し、即時ダメージは「特定の敵兵種」を「ノードを実行した瞬間」に狙い撃ちする。フロー実行中にEND前に敵ユニットを排除できるため戦略的な意味が大きく異なる。弩兵の精密射撃・カウンター能力の割り込みダメージの両方をこの設計で統一する。弩兵の即時ダメージはアーマーを無視してHP直接減算（アーマー貫通）。弩兵の指定兵種が不在の場合は最前列の生存敵ユニットにフォールバックして射撃する（スキップなし）。★v1.37更新*

*※ 史実的根拠：中世クロスボウ（弩）は射速が遅い代わりに貫通力と精度が高く、特定ターゲットを指名して狙う「精密射撃」が主な運用法だった（ジェノバ弩兵による重装騎士の甲冑隙間への射撃）。板金鎧を貫通できた中世最高水準の兵器であり、1139年の教皇インノケンティウス2世による禁止令が発令されるほど凶悪とみなされた。長弓（弓兵）は斉射による面制圧（クレシー・アジャンクールでの陣形全体への矢雨）が主目的であり、狙いを定めない後衛全体への即時均一ダメージとして表現する。ATK蓄積型にすると前段フローの増幅が乗りすぎるため即時ダメージとして切り離した。*

**ループ内の値の扱い（★v1.31確定）**

```
START → [槍兵+3] → [騎士×2] → END
                              ↓
                    攻撃力6が自動確定→敵前衛にダメージ
                    次ループ開始時にリセット
```

- 蓄積値（攻撃力・防御力）はループ内の**中間状態**
- ENDに到達した時点で自動的にダメージ・軽減として確定される
- 次ループ開始時にリセット（重装歩兵の遅延ノードを除く）
- 補給のみループをまたいで持続する

*※ ユニットのHPは引き続き使用する（ユニットの生死判定に必要）。*
*※ 既存の攻撃力・士気パラメータはフロー設計確定後に見直し・再定義する。数値はプロトタイプ検証後に確定。*

## **20.8 補給メカニクス　★v1.30追加**

補給はフローを動かすための燃料リソース。補給が尽きるとフローが止まり、戦闘行動が取れなくなる。

| 項目 | 内容 | 状態 |
| :---- | :---- | :---- |
| 初期値 | 配置ユニット数 × 1 | **確定** |
| 消費 | 1ノード実行 = 補給1消費 | **確定** |
| ループ条件 | 補給が残っている限りフロー末尾から先頭に戻る | **確定** |
| フロー内獲得 | 特定のノードが補給を生産できる（無限ループに近い状態も可能） | **確定（数値未定）** |
| 上限 | 未定 | **未定** |

**補給の設計的意味**

補給が制約になることで「強いノードを全部並べて最適フローを組む」ができなくなる。強力なノードほど補給消費が大きい設計にすることで、フロー設計の中心的なトレードオフが生まれる。

```
例：ユニット4体 → 初期補給4
  [重装歩兵(消費2)] → [騎士(消費2)] → 補給0、ループなし
  [槍兵(消費1)] → [騎士(消費2)] → [弓兵(消費1)] → 補給0、ループなし
  [槍兵(消費1)] → [補給獲得ノード(+2)] → [弓兵(消費1)] → 補給残2、ループ継続
```

*※ 各ノードの補給消費量・補給獲得量はすべて未定。プロトタイプ検証後に確定。*

*※ 史実的根拠：中世軍の兵站（補給線）は作戦継続の生命線だった。補給なしで精鋭を全力投入しても持続できない構造は、百年戦争でのイングランド軍が補給路確保を最優先にした事例（クレシー・ポワティエ遠征）と整合する。*

## **20.10 ユニットの2能力構造　★v1.30追加・v1.31コア能力確定・v1.33更新**

各ユニットはコア能力（固定）とオプション能力（ラン毎変化）の2つの能力を持つ。
